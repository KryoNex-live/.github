# Support

Thank you for your interest in KryoNex.

## Getting Help

If you have questions, need technical assistance, or would like to discuss one of our public engineering projects, we're happy to help.

## Before Contacting Us

Please check the following before opening a support request:

- Project documentation
- Existing GitHub Issues
- Repository README
- Frequently asked questions (if available)

Many common questions may already be answered.

## Types of Support

Depending on the repository, support may include:

- Usage guidance
- Documentation clarification
- Bug investigation
- Feature discussions
- General technical questions

Repository maintainers may define additional support channels within individual projects.

## Business Enquiries

For partnership opportunities, consulting, enterprise solutions, or commercial discussions:

**Email:** support@kryonex.live

## Security Issues

Please do not report security vulnerabilities through public issues.

Refer to our **SECURITY.md** policy for responsible disclosure instructions.

## Community

Constructive discussions, suggestions, and feedback are always welcome.

Thank you for supporting the KryoNex engineering community.
