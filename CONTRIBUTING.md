# Contributing to KryoNex

Thank you for your interest in contributing to KryoNex.

We welcome contributions that improve the quality, reliability, security, and usability of our public engineering resources.

## Ways to Contribute

You can contribute by:

- Reporting bugs
- Suggesting new features
- Improving documentation
- Fixing issues
- Submitting code improvements
- Sharing ideas and best practices

## Before You Start

Before opening a pull request:

- Search existing issues to avoid duplicates.
- Discuss significant changes by creating an issue first.
- Keep pull requests focused on a single objective.
- Follow repository coding standards and documentation guidelines.

## Development Principles

We encourage contributions that align with our engineering philosophy:

- Build maintainable solutions.
- Prioritize security and reliability.
- Write clear, readable code.
- Document significant architectural decisions.
- Prefer simplicity over unnecessary complexity.

## Pull Requests

When submitting a pull request:

- Provide a clear description of the change.
- Reference related issues where applicable.
- Update documentation if required.
- Ensure the project builds successfully.
- Verify that existing functionality is not negatively affected.

## Code Reviews

Every contribution is reviewed before being merged.

Reviews focus on:

- Code quality
- Maintainability
- Performance
- Security
- Documentation
- Overall architectural consistency

Constructive feedback is part of our collaborative engineering process.

## Reporting Security Issues

Please do not report security vulnerabilities through public issues.

Refer to our SECURITY.md policy for responsible disclosure instructions.
