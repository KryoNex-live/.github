# Security Policy

## Security at KryoNex

KryoNex is committed to building secure, reliable, and resilient software. We appreciate the efforts of security researchers and the community in helping us identify and responsibly disclose potential vulnerabilities.

## Reporting a Vulnerability

If you discover a security vulnerability in one of our public repositories, please do **not** report it through public GitHub issues or discussions.

Instead, contact us privately:

**Email:** support@kryonex.live

Please include as much information as possible, including:

- Repository name
- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Proof of concept (if available)
- Suggested remediation (optional)

## Our Commitment

When a valid security report is received, we aim to:

- Acknowledge receipt of the report.
- Review and validate the reported issue.
- Investigate the potential impact.
- Develop and test an appropriate fix.
- Release updates when applicable.
- Keep the reporter informed throughout the process where appropriate.

Response times may vary depending on the complexity and severity of the reported issue.

## Responsible Disclosure

We ask that you:

- Allow us reasonable time to investigate and remediate the issue before public disclosure.
- Avoid accessing, modifying, or deleting data that does not belong to you.
- Avoid actions that could negatively affect users, systems, or services.
- Conduct research responsibly and in good faith.

## Supported Repositories

Security updates and fixes are provided for actively maintained public repositories.

Repository-specific support information may be documented within individual projects.

## Thank You

We value responsible security research and appreciate the efforts of everyone who helps improve the security of KryoNex projects.
